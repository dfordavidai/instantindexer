Setup Guide

1 · Supabase (Database) — 2 min

  1. Go to supabase.com → New Project
  2. SQL Editor → paste schema.sql → Run
  3. Settings → API → copy Project URL + service_role key

2 · Upstash Redis (Queue) — 1 min

  1. Go to upstash.com → Create Database → Redis
  2. Copy the rediss:// URL (used by worker.py on Railway)

3 · Railway (Worker + API) — 3 min

  1. Push worker.py to a GitHub repo
  2. railway.app → New Project → Deploy from GitHub → select repo
  3. Add env vars: UPSTASH_REDIS_URL SUPABASE_URL SUPABASE_SERVICE_KEY INDEXNOW_KEY INDEXNOW_HOST
  4. Start command: python worker.py
  5. Settings → Networking → expose port 8000 → copy your Railway public URL
  6. Check logs for: Worker started — listening on queue: indexing_jobs

4 · Deploy index.html (Any static host) — 1 min

  1. Edit the WORKER_URL constant at the top of the <script> block
  2. Set it to your Railway public URL, e.g. https://indexforce.up.railway.app
  3. Drop index.html on Vercel, Netlify, Cloudflare Pages, or S3 — any static host
  4. No build step. No Node. Just upload the file.

5 · Verify Everything Works

  1. Go to Submit tab → paste 2-3 URLs → Start Indexing
  2. Watch status card: Queued → Processing → Complete
  3. Check Railway logs for the 4 layer completions
  4. After 24h, verification runs automatically and updates Index Rate
💡 Architecture: worker.py runs FastAPI + the queue worker in one process on Railway. index.html calls /api/submit and /api/status/{job_id} directly on Railway. No Next.js. No TypeScript. No build pipeline.
